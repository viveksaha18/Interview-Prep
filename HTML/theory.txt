1. Why it is called mark-up Language?
Ans: HTML uses tags to mark-up different parts of a web page.
Ex <h1>Hello</h1> 
Here h1 tags tells the browser that h1 is a heading

2. What is the purpose of <!Doctype> in html?
Ans: It tells the browser that the document is written in HTML5.

3. What are tags?
Ans: Tags are the pre-defined keywords enclosed in angle brackets.

4. What are attributes?
Ans: Attributes provides extra information about an element. 
Ex: src, alt

5. Difference between id and class?
Ans: id → Used for one unique element on the page.
    class → Used for multiple elements that share the same style or behavior.

6. localStorage aur sessionStorage mein fark?
localStorage ka data permanently store rehta hai (browser band karne ke baad bhi), jab tak manually clear na
ho. sessionStorage ka data sirf tab/window ki session tak rehta hai — tab band karte hi delete ho jata hai.

7. Cookies, localStorage aur sessionStorage mein fark?
Cookies server ko har request ke saath automatically bhejte hain (size limit ~4KB), aur expiry set kar sakte
hain. localStorage/sessionStorage sirf client-side rehte hain, server ko automatically nahi jaate, aur zyada
storage (~5-10MB) allow karte hain.

Q13. <strong> aur <b> mein kya fark hai (same for <em> vs <i>)?
<b> sirf visually bold karta hai, koi semantic meaning nahi. 
<strong> semantically important content
dikhata hai (screen readers isko emphasize karte hain). Same logic 
<i> (visual italic) vs em(semantic emphasis) mein.

Q15. Progressive rendering / DOM ka HTML se kya connection hai?
Browser HTML ko parse karke DOM tree banata hai. CSS se CSSOM banta hai. Dono milkar Render Tree
banate hain jo actual screen par paint hota hai. Yeh flow interviews mein bhi pucha jata hai (critical rendering path)
